# TeamFinder

**TeamFinder** — платформа, на которой разработчики, дизайнеры и другие специалисты находят
единомышленников для совместной работы над pet-проектами. Зарегистрированные пользователи
размещают идеи проектов, собирают команду и откликаются на опубликованные предложения.

> 🔗 **Репозиторий проекта на GitHub:** [https://github.com/ssanqbyte/team-finder](https://github.com/ssanqbyte/team-finder)

## Реализованный вариант задания

**Вариант 1: «Избранное» и фильтрация пользователей** (`TASK_VERSION=1`):

- добавление/удаление проектов в «Избранное» с главной страницы и со страницы проекта;
- страница «Избранное», доступная только её владельцу;
- фильтрация на странице пользователей по 4 критериям: «Авторы избранных проектов»,
  «Авторы проектов, в которых я участвую», «Пользователи, которым нравятся мои проекты»,
  «Участники моих проектов».

## Стек технологий

- Python 3.11+
- Django 5.2
- PostgreSQL 16 (в Docker)
- Pillow (генерация аватарок)
- HTML / CSS / JavaScript (готовые шаблоны)

## Запуск проекта

### Способ 1. Локальный запуск (БД в Docker) — рекомендуемый

1. **Клонируйте репозиторий и перейдите в папку проекта:**
   ```bash
   git clone https://github.com/ssanqbyte/team-finder.git
   cd team-finder
   ```

2. **Создайте и активируйте виртуальное окружение:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate        # Linux/Mac
   venv\Scripts\activate           # Windows
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Создайте файл `.env` по примеру `.env_example`:**
   ```bash
   cp .env_example .env
   ```
   Укажите свой `DJANGO_SECRET_KEY` (можно сгенерировать через
   `python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"`).
   Переменная `TASK_VERSION=1` уже установлена — это реализованный вариант задания.

5. **Запустите PostgreSQL в Docker:**
   ```bash
   docker compose up -d db
   ```
   Данные БД сохраняются в volume `postgres_data` и не теряются при перезапуске.

6. **Примените миграции:**
   ```bash
   python manage.py migrate
   ```

7. **Наполните базу тестовыми данными (рекомендуется для проверки):**
   ```bash
   python manage.py seed_demo
   ```

8. **Запустите сервер разработки:**
   ```bash
   python manage.py runserver
   ```

Проект будет доступен по адресу [http://localhost:8000](http://localhost:8000).

### Способ 2. Полный запуск в Docker Compose

Весь проект (Django + PostgreSQL) можно запустить одной командой:

```bash
docker compose up -d --build
```

Миграции выполнятся автоматически при старте контейнера `web`.
Чтобы загрузить тестовые данные:

```bash
docker compose exec web python manage.py seed_demo
```

Проект будет доступен по адресу [http://localhost:8000](http://localhost:8000).

Остановка: `docker compose down` (данные БД и медиафайлы сохраняются в volumes).

## Функциональность

- **Главная страница** (`/projects/list/`) — карточки проектов (автор, название, описание,
  дата), сортировка от новых к старым, пагинация по 12 проектов, иконка «В избранное»
  и кнопка «Создать проект» для авторизованных.
- **Регистрация** (`/users/register/`) — имя, фамилия, email, пароль; после регистрации —
  переадресация на страницу входа.
- **Вход/выход** (`/users/login/`, `/users/logout/`) — аутентификация по email и паролю.
- **Профиль пользователя** (`/users/<id>/`) — ФИ, аватар, «о себе», контакты, проекты
  пользователя; владельцу доступны кнопки редактирования, добавления проекта и «Поделиться».
- **Редактирование профиля** (`/users/edit-profile/`) — с валидацией телефона
  (форматы `8XXXXXXXXXX` / `+7XXXXXXXXXX`, приведение к `+7...`, проверка уникальности)
  и ссылки на GitHub.
- **Смена пароля** (`/users/change-password/`) — только для залогиненных.
- **Список пользователей** (`/users/list/`) — карточки, сортировка от новых к старым,
  пагинация по 12, фильтры варианта 1.
- **Страница проекта** (`/projects/<id>/`) — информация, участие/отказ от участия,
  для автора — редактирование и завершение проекта, список участников, «Поделиться».
- **Создание/редактирование проекта** (`/projects/create-project/`, `/projects/<id>/edit/`) —
  одна форма, валидация GitHub-ссылки, статус «Открыт»/«Закрыт», редирект на страницу проекта.
- **Избранное** (`/projects/favorites/`) — только для владельца списка.
- **Аватар по умолчанию** — при создании пользователя автоматически генерируется PNG
  с первой буквой имени на случайном приглушённом фоне (Pillow).

## Права доступа

| Роль          | Возможности |
|---------------|-------------|
| Гость         | просмотр проектов, страниц проектов и пользователей, регистрация |
| Авторизованный| + вход/выход, создание/редактирование/завершение своих проектов, участие в чужих, избранное, редактирование профиля |
| Администратор | + управление пользователями и любыми проектами через админ-панель |

## Примечания для ревьюера

1. **Телефон** в модели `User` объявлен с `blank=True`: при регистрации телефон не запрашивается
   (по ТЗ обязательны только имя, фамилия, email, пароль), поэтому на уровне БД поле допускает
   пустое значение. Формат и уникальность телефона проверяются формой редактирования профиля.
2. **После регистрации** пользователь переадресуется на страницу входа — в соответствии с
   текстом ТЗ и чек-листом («После регистрации происходит перенаправление на страницу входа»).
3. **В шаблоны** внесены минимальные изменения: добавлена пагинация
   (`includes/pagination.html`, по 12 элементов), кнопка «Создать проект» на главной показывается
   авторизованным пользователям (по ТЗ), на странице «Избранное» добавлен скрытый блок
   `#empty-favorite-template`, который ожидает готовый JS при удалении последней карточки.
4. **CI**: настроена автоматическая проверка кода (flake8, `manage.py check`) через
   GitHub Actions — `.github/workflows/ci.yml`.
5. Папки `templates_var2` и `templates_var3` удалены, так как реализован вариант 1
   (это допускается стартовым README).
